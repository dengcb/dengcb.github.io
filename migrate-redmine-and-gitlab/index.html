Migrate Redmine and Gitlab

Jan 24 2019 maintain>rails 3 minutes read (About 436 words)

Redmine and Gitlab are great project and code management systems, both based on Ruby on Rails, so I migrate them together and record Rails migration notes here.

Install Rails

  • install RVM in new server

  • install Ruby in new server

  • create new Gemset

Migrate Redmine

  • install dev package needed

    if libjbig0 error in 18.04, download right version
    dpkg -i libjbig0_2.1-3.1build1_amd64.deb

  • modify thin config

    ---
    chdir: /var/www/artec/mine
    environment: production
    address: 0.0.0.0
    port: 3000
    timeout: 30
    log: log/thin.log
    pid: tmp/pids/thin.pid
    max_conns: 1024
    max_persistent_conns: 512
    require: []
    wait: 30
    servers: 5
    daemonize: true

  • migrate Redmine

    • download latest

    • migrate config

    • migrate files and theme

    • install gems, Gemfile add gem "thin", "~> 1.7.2"

Migrate Gitlab

  • install dependency package

    • install nodejs

    • install yarn

    • install go

  • install others

  • Upgrade System

    • upgrade gitlab-shell

    • upgrade Gitaly

  • start migration

  • start checking
